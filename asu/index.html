Coursework

Software Verification, Validation, and Testing

This course stressed the importance of testing early and often, and surveyed a variety of testing technologies, including: model based, combinatorial, mutation and fuzz, static and dynamic, stress, regression, configuration, and others. It went over testing characteristics and management, exit criteria and documentation.

The projects involved getting familiarized with JUnit and Maven to run code coverage and static analysis test suites on code.

Applied Cryptography

First, cryptography was distinguished from steganography on the basis of a publicly available algorithm. We discussed the elements of cryptography, and an overview of the algorithms used to produce nonces, hashes, and encryptions. Finally, these elements are combined in a variety of ways to produce signatures, contracts, money orders, cryptocurrencies, and others. We discussed the vulnerabilities to man-in-the-middle attacks and ways to defend against other attacks.

The projects involved using C# libraries to:

  • Insert hidden messages into images (steganography).
  • Reverse engineer an encryption that uses a flawed key known beforehand to be within a set of time values.
  • Perform a birthday attack to crack an MD5 hash.
  • Encrypt and decrypt Diffie-Hellman and RSA using extended Euclidean.

Information Assurance and Security

This course provided an overview of security policies before detailing the tools used to exploit vulnerabilities in networks, the internet, and software. For example, it describes ICMP flooding, TCP replay attacks, SQL and command injections, cross-site scripting, stack smashing and overwriting the rip register, and taking advantage of insecure scanf and strcpy functions.

The projects involved:

  • Using Python and scapy to spoof an IP address on a VPN, and intercept a UDP message and perform a replay attack to guess the packet ID of a TCP connection using a flawed RNG.
  • Finding vulnerabilities using command injection on a web server to capture flags.
  • Causing a number of programs to crash by hijacking control flows (gdb and Ghidra were used to examine their code and addresses).
  • Writing a dumb fuzzer, written in C to cause a number of programs to crash.
